Hugo多语言博客搭建与维护指南

May 09, 2025
阅读时长: 2 分钟

博客搭建和管理流程

本指南详细记录了如何在 GitHub Pages 上撰写、上传、删除中英文博客的步骤，以及相关 Git 命令。

1. 博客内容撰写（包括中英文）

(1) 创建或修改博客文章

  1. 定位到 Hugo 博客内容目录： 你的博客内容存储在 content 目录下。为了支持中英文内容，你应该在该目录下有相应的 en 和 zh-cn 文件夹。

  2. 创建新文章：

    • 对于英文文章：
      1

      hugo new en/posts/my-new-post.md
    • 对于中文文章：
      1

      hugo new zh-cn/posts/我的新博客.md

  3. 编辑文章： 使用你喜欢的文本编辑器打开并编辑 content/en/posts/my-new-post.md 或 content/zh-cn/posts/我的新博客.md 文件，填写标题、内容等信息。

  4. 添加文章的 Front Matter： 在文章的顶部，确保设置了正确的 Front Matter（包括语言、标题、日期等）：

    1
2
3
4
5
6

    ---
title: "My New Post"
date: 2025-05-09T12:00:00+08:00
draft: true
language: en
---

(2) 本地查看和预览博客：

  • 在项目根目录下，运行以下命令来启动本地开发服务器：
    1

    hugo server -D
    然后，在浏览器中访问 http://localhost:1313 来查看本地预览。

2. 提交和上传博客到 GitHub

(1) 确保所有文件被 Git 跟踪：

  • 使用 git status 查看未提交的文件。
  • 对所有新添加或修改的文件执行 git add：
    1

    git add .

(2) 提交更改：

  • 提交更改并附上合适的提交信息：
    1

    git commit -m "Add new blog post or update"

(3) 推送到 GitHub 仓库：

  • 确保已配置 GitHub 远程仓库（如果未配置，使用以下命令设置远程仓库）：
    1

    git remote add origin https://github.com/KeqiYe/myblog.git
  • 推送到 GitHub 仓库的 main 分支：
    1

    git push origin main
    该操作会触发 GitHub Actions 流程，自动构建博客并将构建结果推送到 GitHub Pages。

3. 删除博客文章

(1) 删除文章文件：

  • 使用 git rm 删除你不再需要的博客文章文件：
    1

    git rm content/en/posts/my-old-post.md
    或者，如果是中文文章：
    1

    git rm content/zh-cn/posts/过时的博客.md

(2) 提交删除操作：

  • 提交删除操作：
    1

    git commit -m "Remove old blog post"

4. 查看和调试部署结果

  • 访问你的 GitHub Pages 地址，检查中英文博客是否正确显示：
    • 英文博客地址：https://keqiye.github.io/en/
    • 中文博客地址：https://keqiye.github.io/zh-cn/

5. 常见问题解决

  • 文章不显示问题：

    • 检查文章的 Front Matter 是否正确设置了 language 字段。
    • 确保所有博客文件已经提交到 GitHub，并且 GitHub Actions 成功执行。

  • 链接错误（例如 example.com/zh-cn/）：

    • 检查 baseURL 配置，确保它是正确的（例如：https://keqiye.github.io/）。
    • 确保部署过程使用了正确的分支和目标仓库。

6. 其他常用 Git 命令

  • 查看当前 Git 状态：

    1

    git status

  • 查看 Git 提交历史：

    1

    git log

  • 撤销已添加的文件（不提交的更改）：

    1

    git restore --staged <file>

  • 恢复未提交的更改（撤销工作区的修改）：

    1

    git restore <file>

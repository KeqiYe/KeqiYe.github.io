SPH基础(五): 自适应时间积分

May 23, 2024
阅读时长: 9 分钟

1. 引言

在流体动力学的数值模拟中，光滑粒子流体动力学（Smoothed Particle Hydrodynamics, SPH）是一种广泛应用的无网格拉格朗日方法。SPH方法的核心之一是对控制方程组进行时间积分，以更新每个粒子的物理状态。

传统的时间积分方案（如简单的欧拉法或固定步长的龙格-库塔法）虽然实现简单，但在处理复杂动态过程时面临挑战：

  • 步长过大可能导致数值不稳定，模拟发散。
  • 步长过小则会极大地增加计算成本，尤其是在模拟过程相对平稳的阶段。

为了在保证计算精度的同时提高效率，自适应步长（Adaptive Time-Stepping） 的积分方法应运而生。本文将详细介绍如何将自适应步长的 Runge-Kutta (RK) 方法应用于 SPH 的计算流程中。

2. SPH 核心方程

在 SPH 中，流体被离散为一系列携带物理属性（质量、密度、速度等）的粒子。其状态演化由一组常微分方程（ODE）描述，主要包括：

  • 连续性方程（密度演化）:

    $$ \frac{d\rho_i}{dt} = \sum_j m_j (\mathbf{v}_i - \mathbf{v}_j) \cdot \nabla_i W_{ij} $$

  • 动量方程（速度演化）:

    $$ \frac{d\mathbf{v}_i}{dt} = -\sum_j m_j \left( \frac{p_i}{\rho_i^2} + \frac{p_j}{\rho_j^2} + \Pi_{ij} \right) \nabla_i W_{ij} + \mathbf{g} $$

其中，i 和 j 是粒子索引，m 是质量，ρ 是密度，p 是压力，v 是速度，Π 是人工粘性项，g 是外力（如重力），W 是核函数。

3. 自适应步长 Runge-Kutta 方法

自适应步长的核心思想是：在每个积分步中，通过比较两种不同精度的计算结果来估计局部截断误差（Local Truncation Error）。根据误差的大小，动态地调整下一步的时间步长 Δt。

一个经典的例子是 RK45（或称 Dormand-Prince、Cash-Karp 方法），它在一个步长内同时计算出一个四阶精度和一个五阶精度的解。

基本逻辑如下:

  1. 用一个 Δt 计算出四阶解 y₄ 和五阶解 y₅。
  2. 计算误差 ε = ||y₅ - y₄||。
  3. 将误差 ε 与预设的容忍度 tol 比较。
    • 如果 ε <= tol：接受本次计算结果（通常使用更高阶的 y₅ 作为最终结果），并可以适当增大大下一轮的步长 Δt。
    • 如果 ε > tol：拒绝本次计算，减小步长 Δt，并用新的小步长重新计算当前步。

步长调整的常用公式为：

$$ \Delta t_{\text{new}} = \Delta t_{\text{old}} \times \text{safe} \times \left( \frac{\text{tol}}{\epsilon} \right)^{p} $$

其中 safe 是一个安全因子（如0.9），p 是一个与方法阶数相关的指数。

3. 自适应步长 Runge-Kutta 方法 (RK45)

为了在 SPH 模拟中实现效率与精度的平衡，我们不能简单地依赖固定步长积分。自适应步长（Adaptive Time-Stepping）方法，特别是基于 Runge-Kutta-Fehlberg 的 RK45 算法，提供了一种优雅的解决方案。它在每个时间步内，通过巧妙地计算两次（一次四阶精度，一次五阶精度），来估计并控制局部截断误差。

3.1 RK45 的核心思想

RK45 的精髓在于“用一次计算，得两个结果”。通过精心选择的一组系数（如 Dormand-Prince 或 Cash-Karp 系数），它可以用6次函数求值（计算k1到k6）同时得到一个四阶精度的解 y_{n+1}^{(4)} 和一个五阶精度的解 y_{n+1}^{(5)}。

RK45 的精髓在于“用一次计算，得两种不同阶数的解”。这依赖于一组精心设计的系数，其中最著名和广泛使用的是 Dormand-Prince 5(4) 对，这也是 MATLAB ode45 的默认选择。

这个方法需要进行 7 次函数求值（计算 k_1 到 k_7），然后用这些 k 的线性组合来构造解。

1. 计算中间斜率 (k_i):

$$ \begin{aligned} k_1 &= f(t_n, y_n) \\ k_2 &= f(t_n + c_2 h, y_n + h(a_{21}k_1)) \\ k_3 &= f(t_n + c_3 h, y_n + h(a_{31}k_1 + a_{32}k_2)) \\ &\vdots \\ k_7 &= f(t_n + c_7 h, y_n + h(a_{71}k_1 + \dots + a_{76}k_6)) \end{aligned} $$

2. 构造解：

  • 五阶解 (更高精度，用于更新状态):

    $$ y_{n+1}^{(5)} = y_n + h \left( b_1 k_1 + b_2 k_2 + b_3 k_3 + b_4 k_4 + b_5 k_5 + b_6 k_6 + b_7 k_7 \right) $$

    在 Dormand-Prince 方法中，为了效率，b_7 被设计为 0，所以 k_7 实际上不参与 y_{n+1}^{(5)} 的计算。

  • 四阶嵌入解 (更低精度，仅用于误差估计):

    $$ y_{n+1}^{(4)} = y_n + h \left( b_1^* k_1 + b_2^* k_2 + b_3^* k_3 + b_4^* k_4 + b_5^* k_5 + b_6^* k_6 + b_7^* k_7 \right) $$

3. 计算误差估计:

局部截断误差 E_{n+1} 可以通过两个解的差来高效计算：

$$ E_{n+1} \approx \left\| y_{n+1}^{(5)} - y_{n+1}^{(4)} \right\| = h \left\| \sum_{i=1}^{7} (b_i - b_i^*) k_i \right\| = h \left\| \sum_{i=1}^{7} e_i k_i \right\| $$

其中 e_i = b_i - b_i^* 构成了误差系数向量。

$$ E_{n+1} \approx \left\| y_{n+1}^{(5)} - y_{n+1}^{(4)} \right\| = h \left\| \sum_{i=1}^{6} (b_i - b_i^*) k_i \right\| $$

3.2 误差控制：模仿 MATLAB ode45 的策略

仅仅得到误差估计是不够的，关键在于如何利用它来判断当前步长 h 是否“好”。一个优秀的误差控制策略不应该只看绝对误差，而应该像 MATLAB 的 ode45 函数一样，同时考虑相对误差 (Relative Tolerance, RelTol) 和绝对误差 (Absolute Tolerance, AbsTol)。

对于状态向量 y 的每一个分量 y_i，我们计算一个容忍度阈值 Tol_i：

$$ \text{Tol}_i = \text{RelTol} \times |y_i| + \text{AbsTol} $$

这个策略的优点在于：

  • 当解 |y_i| 的数值很大时，主要由相对误差 RelTol 控制，保证了误差与解的量级成比例。
  • 当解 |y_i| 的数值很小（趋近于零）时，RelTol \times |y_i| 项会变得过小，此时由绝对误差 AbsTol 托底，防止了对步长的过度严苛要求。

接下来，我们将计算出的误差 E_{n+1} 与这个容忍度 Tol 进行比较。为了对整个向量的误差进行综合评估，我们计算一个标量误差率 err_rate：

$$ \text{err\_rate} = \sqrt{ \frac{1}{N} \sum_{i=1}^{N} \left( \frac{E_{n+1, i}}{\text{Tol}_i} \right)^2 } $$

其中 N 是状态向量 y 的维度。

3.3 步长调整决策

根据计算出的 err_rate，我们做出决策：

  1. 如果 err_rate <= 1.0: 接受当前步。

    • 当前步长 h 是有效的，甚至可能有点过于保守。
    • 我们将更高阶的解 y_{n+1}^{(5)} 作为本次积分的最终结果。
    • 为了提高效率，我们可以尝试在下一个时间步使用一个更大的步长 h_new。

  2. 如果 err_rate > 1.0: 拒绝当前步。

    • 当前步长 h 太大，导致误差超出了容忍范围。
    • 我们丢弃本次计算的所有结果（y_{n+1}^{(4)} 和 y_{n+1}^{(5)}），系统状态回退到 y_n。
    • 我们需要用一个更小的步长 h_new 重新计算当前步。

步长调整的经典公式为：

$$ h_{\text{new}} = h_{\text{old}} \times \text{safe} \times \left( \frac{1.0}{\text{err\_rate}} \right)^{0.2} $$
  • safe: 一个安全因子，通常取 0.8 到 0.9，防止步长调整过于激进。
  • 指数 0.2: 对于一个五阶方法来说，这个值是 1/(k+1)，其中 k=4 是低阶方法的阶数。在实践中，常用的是 1/k_high，即 1/5=0.2。

同时，为了防止步长变化过快，通常会限制步长的最大/最小增长/缩减率（例如，h_new 不得超过 h_old 的5倍，或小于 h_old 的0.2倍）。

// --- 全局参数 ---
double RelTol = 1e-6; // 相对容忍度
double AbsTol = 1e-9; // 绝对容忍度
double h_min = 1e-8, h_max = 1e-2; // 步长上下限
double safe_factor = 0.9;
double max_increase = 5.0, min_decrease = 0.2;

// --- 主循环 ---
double t = 0.0;
double h = initial_dt; // 初始步长

while (t < T_max) {
    bool step_accepted = false;

    while (!step_accepted) {
        if (h < h_min) {
            // 步长过小，可能出现问题
            error("Timestep smaller than h_min");
            break;
        }

        // 1. SPH力计算：根据当前状态 y_n 计算 k1
        //    y_n 包括所有粒子的位置 x 和速度 v
        auto k1 = compute_derivatives(t, y_n);

        // 2. RK45 核心计算：计算 k2, k3, ..., k6
        //    并得到 y_next_4 (四阶解) 和 y_next_5 (五阶解)
        //    (此处省略繁杂的系数计算)
        auto [y_next_4, y_next_5] = perform_rk45_core(t, y_n, h, k1);

        // 3. 计算误差率 err_rate (模仿 MATLAB)
        double err_rate = 0.0;
        for (int i = 0; i < N_particles; ++i) {
            // 对位置和速度分别计算容忍度
            vec3 pos_error = y_next_5.pos[i] - y_next_4.pos[i];
            vec3 vel_error = y_next_5.vel[i] - y_next_4.vel[i];

            vec3 pos_tol = RelTol * abs(y_n.pos[i]) + AbsTol;
            vec3 vel_tol = RelTol * abs(y_n.vel[i]) + AbsTol;

            // 累加误差的平方/容忍度的平方
            err_rate += sum( (pos_error / pos_tol)^2 );
            err_rate += sum( (vel_error / vel_tol)^2 );
        }
        err_rate = sqrt(err_rate / (6 * N_particles)); // 归一化

        // 4. 决策与步长调整
        if (err_rate <= 1.0) {
            // --- 接受步长 ---
            step_accepted = true;
            t += h;
            y_n = y_next_5; // 更新状态为更高阶的解

            // 计算下一个建议步长 (通常是增大的)
            double h_new = h * safe_factor * pow(err_rate, -0.2);
            h = min(h * max_increase, h_new); // 限制最大增幅
            h = min(h, h_max); // 不超过最大步长限制

        } else {
            // --- 拒绝步长 ---
            // 状态 y_n 和 t 保持不变

            // 计算下一个建议步长 (必须是减小的)
            double h_new = h * safe_factor * pow(err_rate, -0.2);
            h = max(h * min_decrease, h_new); // 限制最大降幅
        }
    }

    // (可选) 在每个成功的时间步后，更新邻域、输出数据等
    UpdateAndSaveData(t);
}

// compute_derivatives 函数需要计算SPH的密度、压力、加速度等 // 并返回一个包含所有粒子 d(pos)/dt (=v) 和 d(v)/dt (=a) 的导数向量

SPH控制方程的右端项计算 (Right-Hand Side, RHS)

在SPH（光滑粒子流体动力学）数值模拟中，系统的演化由一组常微分方程（ODEs）描述。使用显式时间积分方案（如Leapfrog或Runge-Kutta）求解这些ODEs的关键，是精确计算每个时间步的右端项（RHS），即各个物理量的时间导数。

以下是SPH中需要求解的核心控制方程及其右端项，特别考虑了包含弹塑性固体力学模型的场景。

1. 动量方程 (Momentum Equation)

方程:

$$ \frac{d\mathbf{v}_i}{dt} = \mathbf{a}_i $$

右端项 (RHS_v): 粒子 $i$ 的加速度 $\mathbf{a}_i$。它由多种力的贡献组成：

$$ \mathbf{a}_i = \mathbf{f}_i^{\text{pressure}} + \mathbf{f}_i^{\text{viscosity}} + \mathbf{f}_i^{\text{deviatoric}} + \mathbf{f}_i^{\text{gravity}} + \dots $$
  • 压力梯度力 (f_pressure):
    1

    f_pressure = - \sum_j m_j \left( \frac{P_i}{\rho_i^2} + \frac{P_j}{\rho_j^2} \right) \nabla_i W_{ij}
  • 人工粘性力 (f_viscosity): 用于处理冲击波。
    1

    f_viscosity = - \sum_j m_j \Pi_{ij} \nabla_i W_{ij}
  • 偏应力梯度力 (f_deviatoric): 此项在固体力学模型中至关重要，代表由剪切等形变引起的力。
    1

    f_deviatoric = \sum_j m_j \left( \frac{\mathbf{S}_i}{\rho_i^2} + \frac{\mathbf{S}_j}{\rho_j^2} \right) \cdot \nabla_i W_{ij}
    其中 $\mathbf{S}$ 是偏应力张量。
  • 体力 (f_gravity, etc.): 如自引力、外加引力场等。

2. 连续性方程 (Continuity Equation)

方程:

$$ \frac{d\rho_i}{dt} = f(\rho)_i $$

右端项 (RHS_rho): 粒子 $i$ 的密度变化率。

$$ \frac{d\rho_i}{dt} = \rho_i \sum_j \frac{m_j}{\rho_j} (\mathbf{v}_i - \mathbf{v}_j) \cdot \nabla_i W_{ij} $$

这也被称为SPH的“求和形式”密度。在许多现代实现中，密度通常不通过积分此方程得到，而是在每个时间步通过直接求和来计算，以保证质量守恒和稳定性。

$$ \rho_i = \sum_j m_j W_{ij} $$

3. 能量方程 (Energy Equation)

方程:

$$ \frac{du_i}{dt} = f(u)_i $$

右端项 (RHS_u): 粒子 $i$ 的比内能变化率。

$$ \frac{du_i}{dt} = \frac{1}{2} \sum_j m_j (\mathbf{v}_i - \mathbf{v}_j) \cdot \left( \left( \frac{P_i}{\rho_i^2} + \frac{P_j}{\rho_j^2} \right) \nabla_i W_{ij} - \Pi_{ij} \nabla_i W_{ij} \right) + \frac{\mathbf{S}_i}{\rho_i^2} : \nabla_i \mathbf{v}_i $$
  • 第一部分: 压力和人工粘性所做的功。
  • 第二部分: 偏应力所做的功，是弹塑性变形中能量耗散和温升的重要来源。

4. 固体力学本构方程 (Constitutive Equations for Solids)

对于弹塑性固体，除了上述流体力学变量，还必须追踪应力张量的演化。这引入了新的ODE。

方程:

$$ \frac{d\mathbf{S}_i}{dt} = f(\mathbf{S})_i $$

右端项 (RHS_S): 粒子 $i$ 的偏应力张量时间导数 (Rate of Deviatoric Stress Tensor)。 为了保证坐标系旋转下的客观性，必须使用客观应力率，如Jaumann率：

$$ \frac{d\mathbf{S}_i}{dt} = 2G \left( \dot{\boldsymbol{\epsilon}}_i - \frac{1}{3}\text{tr}(\dot{\boldsymbol{\epsilon}}_i)\mathbf{I} \right) + \mathbf{S}_i \cdot \boldsymbol{\Omega}_i - \boldsymbol{\Omega}_i \cdot \mathbf{S}_i $$
  • $\dot{\boldsymbol{\epsilon}}_i$: 应变率张量，由速度梯度计算得到。
  • $\boldsymbol{\Omega}_i$: 自旋张量（速度场的反对称部分），用于处理旋转。
  • 注意: 在实际的弹塑性返回映射算法中，这一步通常与塑性校正隐式地结合在一起，而不是直接积分一个显式的导数。

5. 损伤演化方程 (Damage Evolution Equation)

方程:

$$ \frac{dD_i}{dt} = f(D)_i $$

右端项 (RHS_D): 粒子 $i$ 的损伤变量时间导数 (Rate of Damage Variable)。 该方程的形式完全取决于所选的损伤模型。例如，对于一个基于塑性应变累积的简单模型：

$$ \frac{dD_i}{dt} = \frac{1}{\epsilon_f} \frac{d\epsilon_p}{dt} $$
  • $\epsilon_p$: 等效塑性应变。
  • $\epsilon_f$: 材料的断裂应变。
  • 在Grady-Kipp模型中，这个导数的形式会更复杂（如 d(D^(1/3))/dt = ...）。

计算流程概述

在一个典型的RHS函数中，计算顺序至关重要：

  1. 压力计算: 根据密度 ρ 和内能 u，通过状态方程（EoS）计算压力 P。
  2. 屈服模型：区别于常屈服强度，脆性材料如岩石，其屈服强度是压力P的函数。
  3. 总应力：应力通过静水压力（P）和切应力得到。注意需要判断弹性还是屈服，若屈服需使用径向返回算法得到修正切应力。
  4. 粒子循环：循环，累计求和，计算内能，密度，速度，切应力等导数

上述过程中，步骤1，2，3为单粒子计算，4为邻居粒子循环累加。
